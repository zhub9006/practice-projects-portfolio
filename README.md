## 🗂 Repository Structure

```
practice-projects-portfolio/
├── README.md                  ← You are here
├── .gitignore                 ← Excludes build artifacts, env files, etc.
├── docs/
│   └── learning-journey.md    ← Reflective notes on what I've learned
├── projects/
│   ├── 01-calculator/
│   │   └── README.md          ← Project-specific description & instructions
│   ├── 02-todo-list/
│   │   └── README.md
│   ├── ...                    ← (one folder per project)
│   └── 15-recipe-book/
│       └── README.md
```

Each project folder will eventually contain:
- **Source code** (the implementation)
- **README.md** (what the project does, concepts covered, how to run it)
- Any supporting assets (images, data files, config, etc.)

## 💡 Why This Portfolio Exists

- **Practice with purpose** — each project targets specific programming concepts so learning is intentional, not random.
- **Visible progress** — the numbered, status-tracked index makes growth easy to see for me and anyone who visits.
- **Portfolio-ready** — recruiters and hiring managers can quickly scan what I've built and the skills each project demonstrates.
- **Reflective learning** — the `docs/learning-journey.md` file captures insights, struggles, and breakthroughs alongside the code.

---

## 🛠 How to Use This Repo

1. **Pick a project** from the index above that matches your current skill level.
2. **Read its README** to understand the goals and suggested concepts.
3. **Build it** in the project folder using your preferred language/tools.
4. **Update the status** in the main README table (⬜ → 🔄 → ✅).
5. **Reflect** — jot down what you learned (or struggled with) in `docs/learning-journey.md`.
6. **Commit often** — meaningful commit messages tell the story of how you solved problems.
